## Fixed — reconnect lifecycle plan (commits e7a67a8 … 412c5ab)

| # | Finding | Resolution |
|---|---------|------------|
| 1 | Mid-game disconnect broke the game: `handleDisconnect` quit the room unconditionally, slow reconnects were locked out forever, the survivor's `playerIndex` shifted onto the wrong team, and no `gameOver` was ever emitted. | Presence-aware disconnect keeps the seat and emits `presenceChanged`; the opponent may `claimVictory` (reason `forfeit`) after a grace window (`CLAIM_GRACE_MS`, default 60 s). |
| 2 | Reconnect privacy leak: `handleConnect` emitted the full unmasked `GameStateMapper.toDTO`, exposing the opponent's placement and persisted targets. | Reconnect snapshot uses viewer-masked `toDTOForViewer`. |
| 3 | `playerIndex` derived from `room.playersId.indexOf(sessionId)` — index coupling between room membership and `gameState.players`. | Stable player-identity resolution in the coordinators. |
| 4 | Evicted ghost socket stayed in the socket.io room and kept receiving broadcasts. | Eviction now force-disconnects the old socket after `sessionEvicted`. |
| 5 | No turn timer / AFK forfeit (critical for mobile backgrounding). | Per-phase AFK claim (`CLAIM_AFK_LIMIT_MS`, default 120 s); `afkClaimInMs` countdown shipped in game payloads. Voluntary mid-game quit counts as a concession. |

End-to-end smoke-verified 2026-06-12 with two scripted socket.io clients:
disconnect → `presenceChanged`, reconnect → masked snapshot, grace timeout →
`claimVictory` → `gameOver { reason: 'forfeit' }`.

> Frontend note: the reconnect snapshot is pushed **during** the server's
> connection handler — register the `gameStateUpdated` listener before the
> handshake completes or the snapshot is missed.

## Open — hardening items

| # | Finding | Notes |
|---|---------|-------|
| 6 | JWT env unvalidated at boot: missing `JWT_ACCESS_SECRET` / `JWT_ACCESS_EXPIRES_IN` yields tokens signed with `undefined` or without `exp`. Also `.env.example` lists `JWT_REFRESH_SECRET` / `JWT_REFRESH_EXPIRES_IN`, which are unused — refresh tokens are opaque random strings hashed into Postgres. | Apply the fail-fast pattern of `claimConfigFromEnv` (`src/infrastructure/config/claim.config.ts`). |
| 7 | Gateway CORS reads `process.env.CORS_ORIGIN` at decorator-evaluation time, before Nest's `ConfigModule` has loaded `.env`. | Load dotenv before module import (e.g. `import 'dotenv/config'` first in `main.ts`). |
| 8 | Priority queue bound unvalidated: `addEffectsToPriorityQueue` indexes `queue[face.priority + baseSpeed]` into a fixed length-100 queue; an out-of-range template value crashes mid-resolve with a cryptic spread-of-undefined `TypeError`. Current templates max out around 9. | Guard the bound with a descriptive error. |
| 9 | Dockerfile ships devDependencies (copies the build stage's full `node_modules`), and `CMD node dist/main` points at the wrong path — nest builds to `dist/src/main.js`, so the container crashes at start. | Install `--omit=dev` in the production stage; `prisma` CLI must move to `dependencies` for the runtime `migrate deploy`. |

Items 6–9 are being fixed in the commits following this document.
